Layouts

Flat comes with various possible layouts. You can easily change them with just one adjustment.

Default layout

The default layout is: Fluid width, static topbar and static sidebar. You don't need any classes for this layout. However if you want to change it you need to add attributes to the 

<body>
container. Which classes you need for the specific layout is described below. Of course you can combine every layout with eachother which results in an extremly flexable and customizable theme.

Setting the default color

To change the default color for the theme you have to add the classtheme-COLORNAME to the body tag. For example :

<body class="theme-red">
That will make the color to red.

Sidebar hidden

To hide the sidebar by default just add the class nav-hidden to the #content element. 

<div class="container-fluid nav-hidden" id="content">

Sidebar right side

If you want to have the sidebar on the right hand side just use the snippet below. 

<body class="sidebar-right">

Topbar fixed

If you want the topbar fixed at the top you need to use this body element 

<body data-layout-topbar="fixed">

Sidebar fixed

If you want the sidebar fixed you need this 

<body data-layout-sidebar="fixed">

Fixed width

If you want the width of the page to be fixed use this 

<body data-layout="fixed">

Colors

I included 16 predefined colors with the theme. If you want your custom color theme look at the themes.css file and copy the styles which are in for example theme-red selector. Afterwards change the colors and rename the red to an unsed colorname. Feel free to contact me if you need help here!

Disable slide-toggle sidebar for mobile devices

If you want to disable the sidebar you can do this with just one attribute! 

<body data-mobile-sidebar="false">

Typography

This theme uses the default bootstrap typography. I included an demonstration page under the Components navigation. If you want to read more about bootstrap typography follow this link : http://twitter.github.com/bootstrap/base-css.html#typography.

Grid structure

Grid system (bootstrap)

This theme uses the default bootstrap grid-system. Head over to the bootstrap documentation to read more about it: http://twitter.github.com/bootstrap/scaffolding.html#gridSystem.

Widget box

The default widget box has the following structure: 

<div class="box">
  <div class="box-title">
    <h3>
      <i class="icon-reorder"></i>
      <!-- Widget title -->
    </h3>
  </div>
  <div class="box-content">
    <!-- Widget Content -->
  </div>
</div>

Widget small / condensed

Since FLAT v2.0 you can use small widgets and also condensed widgets if you don't want to use the big (default) ones. Simply add the class box-small or box-condensed to the box element.

Widget closed by default

To hide the widget content by default use the following code below and be sure to change the action-arrow in the widget title.

<div class="box-content" style="display: none;"></div>

Widget without padding

Sometimes you want the box content to have no padding. For example with tables. If so, simply add the class nopadding to the box-content container. It should look like this: 

<div class="box">
  <!-- ...box-title... -->
  <div class="box-content nopadding">
    <!-- Widget Content -->
  </div>
</div>

Widget bordered

The bordered box can be done with the class box-bordered on the .box element 

<div class="box box-bordered">
  <!-- Box content -->
</div>

Widget colored

To colorize the widget box add the class box-color to the .box element. 

<div class="box box-color">
  <!-- Box content -->
</div>

Widget specific color

If you don't want the "main" color on the widget box you can use any of the predefined colors. Simply add the colorname as classname. Example (red box): 

<div class="box box-color red">
  <!-- Box content -->
</div>

File structure

CSS Files

The css files are inside the css/ folder. All css files in the css/ folder should be included in every page. The css files from the css/plugins/ folder depends on which plugin you use. I sorted them correctly in their plugin folder to preserve a clean and good overview because it gets really messy if you put all css files in one folder (css/) with that amount of plugins.

Images

Since this is a flat theme it doesn't use really many images. All images are inside the img/ folder. If you want to use retina images simply add the class .retina-ready to the img element and provide an image with image@2x.png (@2x is used for the retina image). Plugin images are inside the img/plugins/ folder and then in their plugin name folder. The folder /img/demo/ contains images which are only for the demonstration. You are not allowed to use these images at your website.

JS Files

Javascript files are sorted like the Images and CSS files. They are inside js/ folder. The javascript plugins are contained in the js/plugins/<plugin name>/ folder. If you need to know which files you need to include please look at the example pages. If you really have no clue what you need to include contact me!

Validation

You don't need any own javascript to validate your forms. Add the class form-validate to you form and use the data-* attributes. Go through the validation page to see some rules or visit the plugins page. 

<form action="#" method="POST" class='form-horizontal form-bordered form-validate'>
  <div class="control-group">
    <label for="t" class="control-label">
      Text input
    </label>
    <div class="controls">
      <input type="text" name="t" id="t" data-rule-required="true" data-rule-minlength="2">
    </div>
  </div>
</form>

Wizard

The class steps-X is used for the number of steps. That is needed to calculate the width of the step-elements correctly. So if you want 2 steps simply use the class steps-2. I predefined 5 classes steps-1..to..steps-5. Every step needs its unique id. Inside the step is the step overview (wizard-steps). If you don't need an overview you can leave it out. Steps can also be validated using the form validation from above.

Range slider

The range slider has various options which can be achieved via html attributes. If you want a slider with 2 handles (ranged slider) you need the attributes data-range, data-rangestart and data-rangestop. The data-rangestart and data-rangestop are the default values of the ranged slider - set the data-range to "true". Other attributes are data-max, data-step and data-min. Those attributes are pretty self explanatory. 

<div class="slider" data-step="25" data-min="0" data-max="250">
  <div class="amount">
  </div>
  <div class="slide">
  </div>
</div>

Username available?

If you want to use the username available form element you have to adjust some things in javascript. In the application.js on line #296 you will find an AJAX request (jQuerys $.post()). You have to adjust the URL which is currently "/check" (uses mockjax to simulate an answer). Your URL should return JSON with the key 'available'. That should be "true" if the username is free and otherwise it will show an error. Of course you could change that behaviour if you have the js skills.

Often your data inside tables has a primary key. The fixed first column is a great feature to sustain the great overview of the table and don't miss the key column. It will be fixed and highlighted. Therefore you can read and find specific rows and columns way faster.

Dynamic ajax loading

This feature will help a lot of developers. There is often a table with just such a huge amount of data that the html is bloated up and the page loading time went down dramatically. With the dynamic ajax loading feature of dataTables this problem is solved! The data can be fetched via javascript, json or a simple txt file. Only the currently visible rows are rendered in html that's why even the sample page with 2500 table records is extremly fast. 

<table  class="table table-nomargin dataTable table-bordered dataTable-scroller">
  <!-- Table content -->
</table>

Pages

Email Template

The email template provides a solid way to inform your clients about any backend actions. Every backend will send emails and therefore this template is a great start. The email templates are based on the HTML Email Boilerplate which you can find here: htmlemailboilerplate.com

If you wonder why that template uses the table-layout style and inline CSS attributes it's fairly simple to explain. Every email client handles incoming emails and their style differently. There are many clients who only handle these old styles and because of that it is always the best to stick to this old layout.

Email template: Images

In order to provide images like I did you need to set a Full-Path to the image. Relative paths won't work (which should be totally as a matter of course). I didn't set full-path because probably many of the buyers will forget to replace my full-path with theirs and then my Server will get traffic from your email readers which could result in a downtime. I wanted to prevent this by not setting a full-path.

Email template Color

I created the email templates in every predefined color to make your life easier. Simply use the color you want and your are ready to go!

Newsletter Email template

The newsletter email template provides a good way to start your own newsletter without much work. The newsletter email template uses Z-Positioned elements. Statistics have shown that Z-Styling is more user-friendly than default I-styling (everything straight below). Therefore that newsletter will be read more frequently and your project/product shines even more!
